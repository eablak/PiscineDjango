How Django Displays a Static Web Page

This is a simplified overview of how Django handles an HTTP request and responds with a static web page, using templates and views.

1. The Request

The process starts when a user sends an HTTP request by entering a URL in their browser. Django receives this request through its URL dispatcher.

2. URL Routing

Django checks the requested URL against the patterns defined in the urls.py file. When it finds a match, it routes the request to the corresponding view function.

3. The View

The view function is responsible for handling the request. For a static page, the view usually calls the render() function to return an HTML response using a template.

4. Template Rendering

Django loads the specified HTML template (like index.html) and renders it. If any context data is passed, it is inserted into the template.

5. The Response

Once the template is rendered into a complete HTML page, Django returns it as an HTTP response. The browser receives and displays the page to the user.

Summary

  • User requests a URL
  • Django matches it in urls.py
  • View function is called
  • Template is rendered
  • HTML is returned as response
